## 三层架构

```
archiki/
├── raw/        ← 原始素材层（只读：原始文章、图片附件）
├── wiki/       ← 编译知识层（AI 维护：建筑师、项目、流派、概念页面）
├── outputs/    ← 生成输出层（AI 生成的回答、年表、分析报告）
├── CLAUDE.md   ← Schema 规则（定义目录结构、页面规范、工作流）
└── README.md
```

## 页面规范

- 每个实体一个原子页面，使用中文名称
- 统一 YAML frontmatter（type、aliases、tags、created/updated）
- `[[wikilink]]` 交叉引用表达实体关系（建筑师 ↔ 项目 ↔ 流派 ↔ 概念）
- 图片使用 `![[image.png]]` 本地嵌入（存放在 `raw/assets/`）
- 四类实体：建筑师 / 项目 / 流派 / 概念

## 实体关系模型

```
建筑师 ──设计──→ 项目
建筑师 ──隶属──→ 流派
流派   ──包含──→ 项目
项目   ──体现──→ 概念
建筑师 ──影响──→ 建筑师
流派   ──影响──→ 流派（继承/反叛）
```
